ケース①：対象 VM が AD 参加していない場合
【1. Hub VNet に Azure DNS Private Resolver 用サブネットを作成】

Azure Portal
  ↓
仮想ネットワーク
  ↓
Hub VNet を選択
  例：vnet-aim-jpe-001
  ↓
サブネット
  ↓
+ サブネット

作成するサブネット：
  名前：
    snet-dnspr-inbound

  アドレス範囲：
    例：10.126.x.0/28

  用途：
    Azure DNS Private Resolver の Inbound Endpoint 用

注意：
  Outbound Endpoint を使わない場合は、
  Inbound 用サブネットだけでよい。

【2. Azure DNS Private Resolver を作成】

Azure Portal
  ↓
DNS Private Resolvers
  または
DNS プライベート リゾルバー
  ↓
作成

基本設定：
  サブスクリプション：
    Hub VNet が存在するサブスクリプション

  リソースグループ：
    DNS管理用のRG
    例：rg-dns-aim-jpe-001

  名前：
    dnspr-aim-jpe-001

  リージョン：
    Hub VNet と同じリージョン

  仮想ネットワーク：
    Hub VNet
    例：vnet-aim-jpe-001

【3. Inbound Endpoint を作成】

Azure DNS Private Resolver 作成画面
  ↓
Inbound Endpoints
  ↓
Add an endpoint

設定内容：
  名前：
    inbound-aim-jpe-001

  サブネット：
    snet-dnspr-inbound

  IP割り当て：
    動的または静的
    ※運用では静的にできるなら静的推奨

用途：
  オンプレDNSから Azure 側の Private DNS Zone を引くための入口。

【4. Private DNS Zone を作成】

Azure Portal
  ↓
Private DNS zones
  または
プライベート DNS ゾーン
  ↓
作成

設定内容：
  サブスクリプション：
    DNS管理用サブスクリプション

  リソースグループ：
    DNS管理用RG

  名前：
    例：azure.internal.example.local
    例：az.corp.local
    例：cloud.internal.local

注意：
  オンプレADのDNSゾーン名と同じ名前は、
  設計上の理由がない限り避ける。

例：
  オンプレAD：
    corp.local

  Azure AD未参加VM用：
    azure.corp.local
    az.corp.local
    cloud.corp.local

【5. Private DNS Zone を Hub VNet にリンク】

Azure Portal
  ↓
Private DNS zones
  ↓
作成したゾーンを選択
  例：azure.internal.example.local
  ↓
Virtual network links
  または
仮想ネットワーク リンク
  ↓
+ 追加

設定内容：
  リンク名：
    link-hub-vnet-aim-jpe-001

  仮想ネットワーク：
    Hub VNet
    例：vnet-aim-jpe-001

  自動登録：
    無効

目的：
  Azure DNS Private Resolver が存在する Hub VNet から、
  Private DNS Zone を解決できるようにする。

【6. Private DNS Zone を Spoke VNet にリンク】

Azure Portal
  ↓
Private DNS zones
  ↓
作成したゾーンを選択
  ↓
Virtual network links
  ↓
+ 追加

設定内容：
  リンク名：
    link-spoke-vnet-xxx

  仮想ネットワーク：
    対象VMが存在する Spoke VNet

  自動登録：
    手動でAレコードを管理する場合：
      無効

    VM名を自動登録したい場合：
      有効

注意：
  自動登録を有効にする場合は、
  そのゾーンをどのVNetの登録用ゾーンにするかを整理してから使う。

【7. Private DNS Zone に A レコードを作成】

Azure Portal
  ↓
Private DNS zones
  ↓
作成したゾーンを選択
  例：azure.internal.example.local
  ↓
Recordsets
  または
レコードセット
  ↓
+ 追加

設定内容：
  名前：
    vm01

  種類：
    A

  TTL：
    3600

  IPアドレス：
    対象VMのプライベートIP
    例：10.126.xx.xx

作成される名前：
  vm01.azure.internal.example.local

登録先IP：
  10.126.xx.xx

【8. DNS Private Resolver Inbound Endpoint 用のNSG設定】

対象：
  snet-dnspr-inbound にNSGを関連付けている場合

許可ルール：
  方向：
    Inbound

  送信元：
    オンプレDNSサーバーのIP
    または
    オンプレDNSサーバーのCIDR

  宛先：
    snet-dnspr-inbound
    または
    Inbound Endpoint のIP

  プロトコル：
    UDP

  ポート：
    53

  アクション：
    Allow

追加ルール：
  プロトコル：
    TCP

  ポート：
    53

  アクション：
    Allow

【9. Hub-Spoke VNet Peering を設定】

Azure Portal
  ↓
仮想ネットワーク
  ↓
Hub VNet
  ↓
ピアリング
  ↓
+ 追加

Hub → Spoke 側：
  仮想ネットワークアクセス：
    有効

  転送されたトラフィックを許可：
    有効

  ゲートウェイ転送を許可：
    Hub の VPN Gateway / ExpressRoute Gateway を Spoke に使わせる場合のみ有効

Spoke → Hub 側：
  仮想ネットワークアクセス：
    有効

  転送されたトラフィックを許可：
    有効

  リモートゲートウェイを使用：
    Hub の Gateway を Spoke から使う場合のみ有効

【10. Spoke VM サブネットに UDR を関連付け】

Azure Portal
  ↓
ルート テーブル
  ↓
作成

設定内容：
  名前：
    rt-spoke-to-onprem

  リージョン：
    Spoke VNet と同じ

作成後：
  ルート
    ↓
  + 追加

ルート設定：
  ルート名：
    to-onprem

  宛先の種類：
    IP アドレス

  宛先IPアドレス/CIDR：
    オンプレCIDR
    例：192.168.0.0/16

  次ホップの種類：
    仮想アプライアンス

  次ホップアドレス：
    Azure Firewall または NVA のプライベートIP

関連付け：
  ルートテーブル
    ↓
  サブネット
    ↓
  関連付け
    ↓
  対象VMが存在する Spoke サブネットを選択

【11. Azure Firewall / NVA で実通信を許可】

対象：
  Hub VNet 上の Azure Firewall
  または
  NVA / 3rd Party Firewall

許可する通信：
  Source：
    オンプレCIDR

  Destination：
    対象VMのプライベートIP
    または
    Spoke VNet / Spoke Subnet のCIDR

  Protocol：
    必要なプロトコル

  Port：
    必要なポートのみ

例：
  RDP：
    TCP 3389

  HTTPS：
    TCP 443

  SMB：
    TCP 445

  WinRM：
    TCP 5985
    TCP 5986

  ICMP：
    必要な場合のみ

【12. 対象VM側のNSGを設定】

Azure Portal
  ↓
仮想マシン
  ↓
対象VM
  ↓
ネットワーク
  ↓
関連付いているNSGを開く
  ↓
受信セキュリティ規則
  ↓
+ 追加

設定内容：
  Source：
    オンプレCIDR

  Source port：
    *

  Destination：
    対象VM
    または
    対象VMのサブネット

  Destination port：
    必要なポート

  Protocol：
    TCP / UDP / ICMP など必要に応じて

  Action：
    Allow

  Priority：
    既存ルールと重複しない番号

  Name：
    allow-onprem-to-vm-required-ports

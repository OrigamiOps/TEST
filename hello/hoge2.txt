ケース②：対象 VM が AD 参加している場合

【1. 対象VMが存在するVNetのDNSサーバー設定】

Azure Portal
  ↓
仮想ネットワーク
  ↓
対象VMが存在する VNet
  ↓
DNS サーバー

設定内容：
  DNSサーバー：
    カスタム

  DNSサーバーIP：
    オンプレAD DNSのIP
    または
    Azure側DNSフォワーダーのIP

例：
  10.x.x.x
  10.x.x.y

目的：
  Azure VM が AD DNS を参照できるようにする。
  AD参加済みVMの名前登録・名前解決をAD DNS側に寄せる。

【2. NIC側に個別DNS設定がある場合は整理】

Azure Portal
  ↓
仮想マシン
  ↓
対象VM
  ↓
ネットワーク
  ↓
ネットワークインターフェイス
  ↓
DNS サーバー

設定方針：
  VNet のDNS設定を使う場合：
    継承設定にする

  NIC個別で指定する場合：
    オンプレAD DNS
    または
    Azure側DNSフォワーダーのIPを指定

注意：
  VNet側とNIC側でDNS設定がバラバラにならないようにする。

【3. Azure Private DNS Zone は原則作成しない】

AD参加済みVMの場合：
  VM名：
    vm01.corp.local

  DNS登録先：
    オンプレAD DNS

Azure側で不要な設定：
  ・Private DNS Zone へのAレコード手動登録
  ・ADドメイン名と同名の Private DNS Zone 作成
  ・Azure DNS Private Resolver Inbound Endpoint 経由のAzure Private DNS参照

注意：
  AD DNSに登録される名前を正とする。

【4. Hub-Spoke VNet Peering を設定】

Azure Portal
  ↓
仮想ネットワーク
  ↓
Hub VNet
  ↓
ピアリング
  ↓
+ 追加

Hub → Spoke 側：
  仮想ネットワークアクセス：
    有効

  転送されたトラフィックを許可：
    有効

  ゲートウェイ転送を許可：
    Hub の VPN Gateway / ExpressRoute Gateway を Spoke に使わせる場合のみ有効

Spoke → Hub 側：
  仮想ネットワークアクセス：
    有効

  転送されたトラフィックを許可：
    有効

  リモートゲートウェイを使用：
    Hub の Gateway を Spoke から使う場合のみ有効



【5. Spoke VM サブネットに UDR を関連付け】

Azure Portal
  ↓
ルート テーブル
  ↓
作成

設定内容：
  名前：
    rt-spoke-to-onprem

  リージョン：
    Spoke VNet と同じ

作成後：
  ルート
    ↓
  + 追加

ルート設定：
  ルート名：
    to-onprem

  宛先の種類：
    IP アドレス

  宛先IPアドレス/CIDR：
    オンプレCIDR
    例：192.168.0.0/16

  次ホップの種類：
    仮想アプライアンス

  次ホップアドレス：
    Azure Firewall または NVA のプライベートIP

関連付け：
  ルートテーブル
    ↓
  サブネット
    ↓
  関連付け
    ↓
  対象VMが存在する Spoke サブネットを選択

【6. Azure Firewall / NVA で実通信を許可】

対象：
  Hub VNet 上の Azure Firewall
  または
  NVA / 3rd Party Firewall

許可する通信：
  Source：
    オンプレCIDR

  Destination：
    対象VMのプライベートIP
    または
    Spoke VNet / Spoke Subnet のCIDR

  Protocol：
    必要なプロトコル

  Port：
    必要なポートのみ

例：
  RDP：
    TCP 3389

  HTTPS：
    TCP 443

  SMB：
    TCP 445

  WinRM：
    TCP 5985
    TCP 5986

  AD関連通信が必要な場合：
    DNS TCP/UDP 53
    Kerberos TCP/UDP 88
    LDAP TCP/UDP 389
    LDAPS TCP 636
    SMB TCP 445
    RPC TCP 135
    動的RPCポート

【7. 対象VM側のNSGを設定】

Azure Portal
  ↓
仮想マシン
  ↓
対象VM
  ↓
ネットワーク
  ↓
関連付いているNSGを開く
  ↓
受信セキュリティ規則
  ↓
+ 追加

設定内容：
  Source：
    オンプレCIDR

  Source port：
    *

  Destination：
    対象VM
    または
    対象VMのサブネット

  Destination port：
    必要なポート

  Protocol：
    TCP / UDP / ICMP など必要に応じて

  Action：
    Allow

  Priority：
    既存ルールと重複しない番号

  Name：
    allow-onprem-to-adjoined-vm-required-ports
